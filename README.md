# SaaS-Task (Test)

## Описание
Пример реализации биллинговой системы для SaaS-приложения.
Реальная интеграция с банком заменена возможной мок-реализацией.
В качестве примера используется Т-Банк.

## Архитектура
- Node.js + Express
- REST API
- Webhook для обработки статуса платежей

## Сценарий оплаты со стороны пользователя
1. Клиент инициирует оплату
2. Создается счёт
3. Создается платёж
4. Пользователь перенаправляется на страницу банка
5. Банк присылает webhook
6. Статус платежа обновляется

## Возможные риски
- Повторные webhook
- Потеря ответа от банка
- Несовпадение суммы
- Атаки на webhook

## Защита
- Уникальные ключи
- Транзакции (например, подтверждение платежа и обновление счета)
- Идемпотентная обработка webhook
